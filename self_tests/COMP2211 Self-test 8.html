Self-test 8:
Digital Image Processing Fundamentals

  1. (Simple) An image has dimensions 100×200. What is the image coordinate of the bottom-right most pixel?

    1. (100,200)
    2. (200,100)
    3. (99,199)
    4. (199,99)
    Solution
    The answer is C. Dimensions of 100×200 means the width of the image is 100, and the height is 200. Since image coordinates starts from (0,0), the bottom-most pixel should be (99,199).

  2. (Simple) Let I be the pixel intensity for a pixel of a grayscale image. Consider applying the following operation to each pixel: Inew=floor(0.5×Iold) Which of the following is correct?

    1. The resulting image will be pure black everywhere.
    2. The resulting image will be darker.
    3. The resulting image will be brighter.
    4. No image can be formed.
    Solution
    The answer is B. A grayscale image has intensities in [0,255]. Dividing by 2 and then rounding down makes each pixel's intensity closer to 0, so the overall image is darker.

  3. (Medium) Let P(x,y) denote the pixel intensity value at the image coordinate (x,y). After applying the following transformation, the new pixel intensity at location (x,y), denoted Pnew(x,y), is given by Pnew(x,y)=P([101011][xy1]) Describe in words what transformation was applied.

    Solution
    A translation upwards and leftwards by 1 unit was applied. Expanding out the matrix multiplication gives Pnew(x,y)=P(x+1,y+1). So the new intensity at (x,y) is equal to the intensity of the pixel one unit downwards and one unit rightwards.

  10. (Simple) What is the advantage of using Otsu's Method for image thresholding compared to the regular image thresholding algorithm?

    Solution
    The resulting threshold value is reproducible. Given the same image, two researchers using Otsu's algorithm must arrive at the same threshold T.

    Input parameters:

    • image : A NumPy array with shape M×M×D
    • kernel : A NumPy array with shape N×N×D
    • return: A NumPy array of the resulting image

    Note that:

    • You may assume stride = 1
    • You may assume M≥N
    • No padding is necessary
    • No need to flip the kernel
    • You are allowed to use all functions under the NumPy library, and any built-in python functions
    • You are NOT allowed to add any more explicit loops. This includes for loops, while loops, and list comprehensions
    • You are NOT allowed to use OpenCV

    Solution 
    import numpy as np
def convolve(image, kernel):
    # get the dimensions of the image, kernel, and number of channels
    M = image.shape[0]
    N = kernel.shape[0]
    channels = image.shape[2]

    # construct an empty NumPy array, which will store the convolved image
    result_dim = M - N + 1
    result = np.zeros((result_dim, result_dim, channels), dtype=np.float32)
    
    # perform convolution
    for x in range(0, result_dim):
        for y in range(0, result_dim):
            result[y, x] = np.sum(image[y:y+N, x:x+N] * kernel, axis=(0, 1))
    
    return result
