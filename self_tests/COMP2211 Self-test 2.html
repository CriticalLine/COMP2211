  15. Suppose we have an array M of shape (22, 11). Which of the following operations are valid? Also, determine the shape of the array after the operation.

    From your TA: I don't know about you, but argsort is super hard for me to understand at first, and the documentation is quite unclear. I will throw in a bit of my own explaination. The argsort function returns an array of "rankings". The first element in np.argsort(array) returns the *index of* the smallest element in the array. The second element in np.argsort(array) returns the *index of* the second smallest element in he array, and so on.   29. What is the output of the following code? 

      import numpy as np

  haha = np.array([1, 2, 3])
  hehe = haha
  hehe[2] = 999
  print(haha)
    Solution

    [1 2 999]. There are two properties of python and numpy working together:

    1. Python grabs things by reference: In the statement hehe = haha, we are not copying the values of haha to hehe; instead, we are saying that hehe should point to whatever memory location that haha is pointing to.

    2. NumPy arrays are mutable: In the statement hehe[2] = 999, we can directly change the value of the numpy array that hehe is pointing at. Since hehe and haha are pointing to the same array by property (1), the array that haha points to is precisely the one that got changed.

    We will split the code into multiple parts as follows to facilitate explanation. First, convince yourself that the following code snippet is functionally identical to the one above. 
      aT = a.reshape(num_datas, 1)

    The flags array is essentially what we need, except in boolean. We only need to turn all the True's to 1's and False's to 0's. The last line is just one of the many ways to achieve this result.

Suppose an array X of shape (2211, 3) holds 2211 data points (which is our training data set), and another array T of shape (300, 3) holds 300 data points (which is our test data set). Using broadcasting magic, complete the following code snippet that computes the NumPy array "dists" of shape (2211, 300), where dists[i, j] is the distance between X[i] and T[j] for each i and j.

    For the sake of terminology, the distance between x = (x1, x2, x3) and y = (y1, y2, y3) is defined as the square root of (x1 - y1)^2 + (x2 - y2)^2 + (x3 - y3)^2 

      import numpy as np
  from dataset import training_data, test_data

  X, T = training_data, test_data
  print(X.shape, T.shape)
  
      # Print "(2211, 3) (300, 3)"
  
  # Your code goes here
    
  print(dists.shape)
  
      # Print "(2211, 300)"
    Your code should be at most five lines, and should not contain any loops. Try not to hard-code the numbers "2211" and "300" into your code.
    Solution

    This is actually a question from PA1 and the midterm of Spring 2022. We provide two solutions for this question, and we shall try our best to explain both. You have to understand at least one of them.

    Solution 1: 

      X_reshaped = X.reshape(-1, 1, 3)
  T_reshaped = T.reshape(1, -1, 3)
  dists = np.sqrt(np.sum((T_reshaped - X_reshaped) ** 2, axis = 2))
    This solution is perhaps best understood via a diagram.

    From the diagram, we can see that the array T_reshaped - X_reshaped has shape (2211, 300, 3), and the [i,j]-th row of the array is the element-wise subtraction of T[i] and X[j]. Then, we square it element-wise, add up the square of each coordinate, and take the square root.
    Note 1: This solution seems to have the drawback of having to hard-code the number of features compared to solution 2; one way to fix this is via the following 
      num_features = X.shape[-1]
  X_reshaped = X.reshape(-1, 1, num_features)
  T_reshaped = T.reshape(1, -1, num_features)
  dists = np.sqrt(np.sum((T_reshaped - X_reshaped) ** 2, axis = 2))
    Note 2: In the official marking scheme, this solution is further reduced into a cryptic one-line solution in two different ways under the "alternative solutions" section: 
      dists = np.sqrt(np.sum(np.square(np.expand_dims(X_test, 1) - X_train), axis=2))
  dists = np.sqrt(np.sum(np.square(X_test[:,None] - X_train[None]), axis=-1))
    but they are essentially the same as our solution here.

    Solution 2: 
      M = -2 * np.dot(T, X.transpose())
  sum_T_squared = np.square(T).sum(axis=1)
  sum_X_squared = np.square(X).sum(axis=1)
  dists = np.sqrt(M + sum_T_squared.reshape(-1, 1) + sum_X_squared.reshape(1, -1))
    This solution uses the expansion formula (a - b)^2 = a^2 - 2ab + b^2. It is essentially this formula on the scale of an entire NumPy array.

    M represents the "-2ab" part. Recall that np.dot() is matrix multiplication on 2D arrays, so np.dot(T, X.transpose())[i, j] is the sum of the coordinate-wise product of T[i] and X[j]. It has shape (2211, 300).
    sum_T_squared represents the a^2 part. Recall that np.square(T) computes the element-wise square of T, so it has shape (2211, 3). Summing it along axis 1, we get an array of shape (2211,), in which the i-th entry represents the sum of squares of each data point. Similarly, sum_X_squared has shape (300,)
    In the last line, we make the arrays into the correct shape, which sets us up for adding everything using broadcasting magic twice, completing the task.
